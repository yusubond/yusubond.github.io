  2. Java语法：import

    3. import:导入包中的类，语法如下：

    import package1.package2.classname;

    package为包名，classname为类名。例如：

    import java.util.Date;     //导入java.util包下的Date类
    import java.swing.*;       //导入java.swing包下的所有类

    注意:import只能导入包所包含的类，而不能导入包。

  5. Atom添加Git Plus插件

    6. Git Plus插件可实现在Atom编辑器里做git things，而不用切换至terminal。
    注意：本文基于ubuntu环境系统。安装步骤如下：

    • 配置user.name和user.email比较简单不做介绍。
    • 生成新的SSH Key，并添加到ssh-agent。可参考Git Help

      • 验证是否添加成功可以在终端执行ssh -T git@github.com进行查看。
    • 安装Git Plus package。在Atom->Settings,install项里面，搜索git-plus，安装即可。
    • 安装完成后，即可在Packages->Git Plus里面使用相应的git指令，亦或使用Ctrl-Shift-H快捷键，做一些git things。

    原文地址：git-plus

    在这个过程中可能遇到的问题及解决方法，如下所示：

    问题１：执行git push后，报错：fatal: could not read Username for 'https://github.com': Device not configured

    原因就是不能识别远端的用户名，解决方法就是重新设置远程地址即可。git remote set-url origin git@github.com:{yourname}/{your_repo_name}.git

    另外在Git 2.0中push有两种模式可选matching和simple。matching模式是提交名字一样的分支，所以可能的结果就是把其他分支的代码一起提交了，而这可能不是你想要的。而simple是push当前分支的代码。
    其设置命令如下:
    git config --global push.default simpleorgit config --global push.default matching

  6. Linux系统4G上网卡的使用配置

    7. 1、Linux系统下通过修改内核设置usbserial模式

    2、利用usb_modeswitch工具切换usb工作模式，其配置文件为/etc/usb_modeswitch.conf

    3、利用wvdial工具配置上网卡参数，其配置文件为/etc/wvdial.conf

    4、启用上网卡

    sudo insmod /lib/modules/$(uname -r)/kernel/drivers/usb/serial/usbserial.ko
sudo usb_modeswitch -W -c /etc/usb_modeswitch.conf
sudo wvdialconf /etc/wvdial.conf
sudo wvdial

    9. 在编程oai-mme和oai-spgw时，因为没有libgtpnl头文件而导致编译失败，可自行安装libgtpnl包，如下所示：

    git clone git://git.osmocom.org/libgtpnl

    创建编译目录：
    cd libgtpnl
    mkdir build-aux

    之后就是安装

    sudo apt-get install libmnl-dev  #this will be needed when make
aclocal
autoreconf --install
automake --add-missing
./configure
make
sudo make install

  10. Ubuntu启动问题以及Grub Rescue修复方法

    11. 在安装双系统时，或者不小心删除了启动引导项时，启动时系统将进入Grub Rescue模式。在rescue模式下，只用少量的命令可以使用，但是通过一定的操作可以让系统加载正常模块，然后进入正常启动模式，启动后修复grub启动项即可。

    rescue模式下可使用的命令有：set,ls(列出文件),insmod(插入启动模块),root(设置启动分区),prefix(设置启动路径)。

    解决方法如下：

    # 查看系统分区情况
grub rescue>ls
# 罗列出所有的磁盘分区信息，比如说：
(hd0,msdos6) (hd0,msdos5) (hd0,msdos4) (hd0,msdos3)(hd0,msdos2)
# 查找曾经安装系统的磁盘分区，找到相应的grub
grub rescue>ls (hd0,X)/        # 查看X磁盘分区/根路径的文件信息
grub rescue>ls (hd0,X)/boot    # 查看X磁盘分区/boot路径的文件信息
grub rescue>ls (hd0,X)/grub    # 查看X磁盘分区/grub路径的文件信息
# 假设找的(hd0,msdos3)时，显示了文件信息，则表明系统安装在这个分区，并记住相应的grub路径
# 接下来，设置系统的启动项，调用如下命令
grub rescue>set root=(hd0,msdos3)
grub rescue>set prefix=(hd0,msdos3)/grub  #这个路径为之前找到的启动项路径
grub rescue>insmod normal
# 设置完成后，进行启动即可
grub rescue>normal

    经过以上操作，就可以显示丢失的grub菜单，然后进入自己想进入的系统即可。

    注意：如果进行重启，问题依旧存在。所以最关键的一步就是对grub进行修复。进入系统后，使用以下命令对grub进行修复。

    sudo update-grub
sudo grub-install /dev/sda
# sda是磁盘号码，不是分区号码

    grub修复成功后，重启测试即可。说明一下，grub修复成功后，其启动项是由当前系统去引导另一系统启动。你也可以进入另一系统进行grub修复，修改启动引导顺序，或者修改文件/etc/default/grub设定默认启动项(GRUB_DEFAULT从0开始计算)。
